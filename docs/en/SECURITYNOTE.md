# MindSpeed Security Statement

## System Security Hardening

- You are advised to enable **address space layout randomization** (ASLR) level 2 in the system.
Run the following command to enable it:

  ```bash
  echo 2 > /proc/sys/kernel/randomize_va_space
  ```

## Running User Recommendations

- For security considerations, it is recommended that you avoid using administrator-type accounts such as root when executing any commands, and follow the principle of least privilege.

## File Permission Control

- It is recommended that users set the system umask value to 0027 or higher on the host (including the physical machine) and in containers, ensuring that the default maximum permission for newly created folders is 750 and the default maximum permission for newly created files is 640.
- It is recommended that users implement security measures such as permission control for files required for training, files saved during training, users' personal privacy data, commercial assets, and other sensitive files. For example, write permission control for dataset files in multi-user shared dataset scenarios. The recommended permissions can be set by referring to Table 1.
- Various fusion operators in MindSpeed are compiled by calling the cpp_extension feature in PyTorch. The compilation results are cached by default in the `~/.cache/torch_extensions` directory. It is recommended that users apply permission control to the generated files as needed by referring to Table 1.
- The file permissions for files generated during the operation of native Megatron-LM and the PyTorch framework depend on system settings, such as dataset index files generated by Megatron-LM and files saved via the torch.save API. It is recommended that users executing the current script apply appropriate permission control to the generated files based on their own needs, and the permissions can be set by referring to Table 1.
- Permission control must be implemented during user installation and usage. It is recommended to refer to Table 1 for settings. If installation/uninstallation logs need to be saved, the parameter `--log <FILE>` can be appended to the installation/uninstallation command. Ensure that permission control is applied to the `<FILE>` file and its directory.

### Table 1 Recommended maximum values for file (folder) permission control in various scenarios

| Type           | Linux Permission Reference Maximum |
| -------------- | ---------------  |
| User home directory                        |   750 (rwxr-x---)            |
| Program files (including scripts and libraries)       |   550 (r-xr-x---)             |
| Program file directory                      |   550 (r-xr-x---)            |
| Configuration files                          |  640 (rw-r-----)             |
| Configuration file directory                      |   750 (rwxr-x---)            |
| Log files (recorded or archived)        |  440 (r--r-----)             |
| Log files (being recorded)                |    640 (rw-r-----)           |
| Log file directory                      |   750 (rwxr-x---)            |
| Debug files                         |  640 (rw-r-----)         |
| Debug file directory                     |   750 (rwxr-x---)  |
| Temporary file directory                      |   750 (rwxr-x---)   |
| Maintenance and upgrade file directory                  |   770 (rwxrwx---)    |
| Service data files                      |   640 (rw-r-----)    |
| Service data file directory                  |   750 (rwxr-x---)      |
| Key components, private keys, certificates, and ciphertext file directory    |  700 (rwx------)      |
| Key components, private keys, certificates, and ciphertext files        | 600 (rw-------)      |
| APIs and scripts for encryption and decryption            |   500 (r-x------)        |

## Data Security Statement

- MindSpeed relies on the basic capabilities of CANN to implement functions such as AOE performance tuning, operator dump, and log recording. Users need to pay attention to the permission control of files generated by the above functions.

## Runtime Security Statement

- It is recommended that users write corresponding training scripts based on the resource conditions of the operating environment. If the training script does not match the resource conditions, such as the dataset loading memory size exceeding the memory capacity limit, or the training script generating local data exceeding the disk space size, errors may be triggered and cause the process to exit unexpectedly.
- When an exception occurs during runtime, MindSpeed will exit the process and print an error message. It is recommended to locate the specific error cause based on the error message, including methods such as setting operator synchronous execution, checking CANN logs, and parsing generated Core Dump files.
- MindSpeed may call the `torch.load` function during runtime. For versions below 2.6, the default parameter of `torch.load` is `weight_only=False`, which poses a potential security risk (CVE-2025-32434). It is recommended to use PyTorch version 2.6.0.
- During the execution of MindSpeed, model loading and saving operations may be performed. Special attention should be paid to the fact that its underlying implementation may use the Python pickle module for serialization/deserialization of some files, and this module has inherent security risks.

## Open API Statement

- MindSpeed has updated its API policy. In addition to seamless support for native Megatron on Ascend devices, it now also adds open APIs for fused operators. Users can directly call these newly added fused operator APIs to fully leverage MindSpeed's optimization capabilities for specific computing tasks.

### Determining Whether a Function Is an Open API

If a function is defined in `__all__` and documented as an external API in `MindSpeed/tree/{branch}/docs/en`, it is considered an open API and can be relied upon as a public function. For specific usage methods and scenarios of this external API, please refer to the API user manual in `docs/en`. If you need to rely on a function that is not documented, please open an Issue on the MindSpeed homepage to confirm with us whether the function is an open API, whether it was exposed accidentally, or whether it may be removed in the future.
